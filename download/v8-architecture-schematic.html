Key Architectural Differences — v7.0 Monolith → v8.0 Tripartite

v7.0 — Monolithic Skill

✕
Single SKILL.md with all modules mixed together (~600 lines)
✕
No intent routing — entire skill loads for every query
✕
Tailwind v3 config pattern (tailwind.config.js)
✕
14 components only, 6+ missing (Textarea, Select, etc.)
✕
Data claims vs reality gap (48 palettes/12 actual)
✕
forwardRef pattern (deprecated in React 19)
✕
No APCA contrast — WCAG 2.x only
✕
No ModalStackProvider for nested dialogs
✕
Basic search.py — no domain aliases, no column search
✕
No @starting-style, @scope, @property coverage

v8.0 — Tripartite Architecture

✓
3 independent skills: A (Infra) + B (Components) + C (Data)
✓
Silent Protocol router with 3 diagnostic questions per query
✓
Tailwind v4 @theme in CSS — no config.js needed
✓
25+ components with full React 19, a11y, dark mode support
✓
1,321 verified records across 24 data files with JSON index
✓
ref as regular prop — no forwardRef wrapper
✓
Dual validation: WCAG 2.x (compliance) + APCA (future-proof)
✓
ModalStackProvider with aria-hidden management for nesting
✓
lookup.py with domain aliases, column search, fuzzy match, JSON output
✓
Full CSS primitives: @starting-style, @scope, @property, anchor(), View Transitions
v8.0 Resolves All 24 Audit Issues from v7.0
HIGH #1
Broken data references (48 claimed / 12 actual)
Part C now has 96 verified palettes, 67 styles, 56 fonts across 24 CSV files with JSON index.
HIGH #2
Missing components (6+ gaps)
Part B now ships 25+ components including Select, Checkbox, Switch, Textarea, Form, PasswordInput, RadioGroup, CommandPalette.
HIGH #3
Tailwind v3 patterns in v4 era
Part A Module 2.7 fully documents @theme migration, token mapping, and v3→v4 upgrade path.
HIGH #4
Modal a11y gaps (no aria-hidden for nesting)
Part B Module 4.3 adds ModalStackProvider that manages aria-hidden across nested dialogs.
HIGH #5
No APCA contrast algorithm
Part A Module 2.6 implements dual validation: WCAG 2.x for compliance + APCA Lc values for future-proofing.
HIGH #6
forwardRef in React 19
All Part B components use ref as regular prop pattern, compatible with React 19 ref forwarding.
RESOLVED
Skeleton aria-hidden contradicts role="status"
Fixed: removed aria-hidden from Skeleton. role="status" is a live region that must be visible to screen readers.
RESOLVED
Missing GSAP plugin registration
Fixed: ImageReveal now calls gsap.registerPlugin(ScrollTrigger) before use.
